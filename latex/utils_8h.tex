{\footnotesize\ttfamily \+\_\+\+\_\+device\+\_\+\+\_\+ double GPU\+\_\+interp\+\_\+scalar (\begin{DoxyParamCaption}\item[{cuda\+Texture\+Object\+\_\+t}]{var,  }\item[{const int}]{mmenemonics,  }\item[{const int}]{i,  }\item[{const int}]{j,  }\item[{const int}]{k,  }\item[{const double}]{del\mbox{[}4\mbox{]} }\end{DoxyParamCaption})}



Interpolates a scalar field using hardware-\/accelerated CUDA Texture Objects. 

This function retrieves the value of a physical variable at a sub-\/grid location. Unlike {\ttfamily GPU\+\_\+interp\+\_\+scalar\+\_\+pointer}, which performs a manual 8-\/point trilinear interpolation in software, this version uses the GPU\textquotesingle{}s dedicated texture mapping units (TMUs) to accelerate the lookups.

\begin{DoxyNote}{Note}
To handle 4D data (3 spatial dimensions + multiple physical variables), the spatial index \textquotesingle{}k\textquotesingle{} and the variable index \textquotesingle{}mmenemonics\textquotesingle{} are flattened into the texture\textquotesingle{}s X-\/axis.

By fetching two discrete points, {\ttfamily k $\ast$ NPRIM} and {\ttfamily (k+1) $\ast$ NPRIM}, and blending them with {\ttfamily del\mbox{[}3\mbox{]}}, we ensure we are only interpolating the {\itshape same} variable across two different spatial slices.
\end{DoxyNote}

Performs trilinear interpolation of a scalar field at a sub-\/grid location. 

This function calculates the value of a specific fluid variable at an arbitrary position within a 3D grid cell. It uses eight precomputed weights (coefficients) and the values at the eight surrounding grid nodes.
